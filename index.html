WHY THIS VERSION WORKS

Static frontend + serverless backend.

Your frontend stays simple with plain HTML, CSS, and JS. The live data calls happen through Vercel serverless functions in the api folder, so your keys stay hidden.

WHAT YOU ADD IN VERCEL

Just environment variables.

No package.json, no build command, no weird framework config. Upload the files, add your keys in Vercel settings, and redeploy.

FAQ

Quick shit you should know

Do I need React, Vite, or npm?

No. This setup is plain HTML, CSS, JS, and Vercel API routes.

Are the keys visible in the browser?

No. The browser only calls your own /api/... routes. Put your real keys in Vercel environment variables only.

Will this work on a normal Vercel static deployment?

Yes, as long as the files are in the repo root and the api folder is there too.
